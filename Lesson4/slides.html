📱 客户端（Client）是什么？

就是你正在用的那个程序

  • 浏览器（Chrome、Safari、Edge）
  • 手机 App（微信、抖音）
  • 桌面软件（QQ、网易云音乐）

它负责……

  • 显示 把界面画给你看
  • 收集 接住你的点击、输入
  • 沟通 把你的需求发送给服务器

🖥️ 服务器（Server）是什么？

放在远处的一台特殊电脑

  • 一直开着，等着大家来访问
  • 有一个固定的「地址」（IP / 域名）
  • 同时为很多客户端服务

它负责……

  • 计算 处理你交来的请求
  • 存储 把数据记下来（账本）
  • 共享 让所有人看到同一份数据

🧋 用「点奶茶」来类比

🙋

顾客

= 客户端
看菜单、下单、取奶茶

📝

记账、做奶茶、管库存

他们只通过一张小票交流——这就是「接口（API）」。

1

搭最简 Express 服务器

⏱ 约 10 分钟 · 目标：访问 http://127.0.0.1:3000 看到问候


// server.js
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello, 投票！');
});

app.listen(3000, () => {
  console.log('服务器跑起来啦：http://127.0.0.1:3000');
});

运行：node server.js

2

读 books.json，暴露 /api/books

⏱ 约 15 分钟 · 目标：浏览器访问 /api/books 看到 JSON 数据


const express = require('express');
const fs = require('fs');
const app = express();

app.get('/api/books', (req, res) => {
  const raw = fs.readFileSync('./books.json', 'utf-8');
  const books = JSON.parse(raw);
  res.json({ ok: true, data: books });
});

app.listen(3000);
💡 改改 books.json 里的票数，刷新看变化——这就是"服务器从文件读数据"

Step 3 · 前端拉数据 + 渲染


// public/app.js
async function loadBooks() {
  const res = await fetch('/api/books');
  const { data } = await res.json();
  const list = document.getElementById('book-list');
  list.innerHTML = data.map(b => `
    <div class="card">
      <h3>${b.title}</h3>
      <p>${b.author} · ${b.desc}</p>
      <p>🗳 <span class="votes">${b.votes}</span> 票</p>
      <button data-id="${b.id}">投一票</button>
    </div>
  `).join('');
}
loadBooks();

4

投票接口 + 按钮交互

⏱ 约 20 分钟 · 后端新增 POST 路由


app.use(express.json());   // 解析 JSON 请求体

app.post('/api/vote', (req, res) => {
  const { bookId } = req.body;
  const books = JSON.parse(fs.readFileSync('./books.json', 'utf-8'));
  const book = books.find(b => b.id === bookId);
  if (!book) return res.json({ ok: false, error: 'book not found' });

  book.votes += 1;
  fs.writeFileSync('./books.json', JSON.stringify(books, null, 2));
  res.json({ ok: true, bookId, votes: book.votes });
});
📖 投完票后，打开 books.json 给学员看 —— 数据真的被改了！

Step 4 · 前端发起 POST


document.getElementById('book-list').addEventListener('click', async (e) => {
  if (e.target.tagName !== 'BUTTON') return;
  const bookId = Number(e.target.dataset.id);

  const res = await fetch('/api/vote', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ bookId })
  });
  const data = await res.json();

  if (data.ok) {
    // 更新页面上的票数
    e.target.closest('.card').querySelector('.votes').textContent = data.votes;
  }
});

🤔 我们的方案有什么局限？

局限说明真实世界的解法
清缓存就能再投 localStorage 只在自己浏览器里 登录系统（账号绑定）
多人同时写文件可能冲突 JSON 文件无并发保护 数据库（事务、锁）
书单写死在代码里 改书要重启服务器 管理后台 + 数据库

这些都是真实工程要解决的问题——今天先打地基。
