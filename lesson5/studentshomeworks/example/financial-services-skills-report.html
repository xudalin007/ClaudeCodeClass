仓库概览

该仓库提供 9 个端到端 Agent（如 Pitch Agent、Earnings Reviewer、GL Reconciler 等）和 9 个按业务线分组的 Vertical Plugin。Agent 是自包含的工作流；Vertical Plugin 是可单独安装的能力集合。所有 Skill 既可通过 Claude Cowork 插件安装，也可经 Claude Managed Agents API 部署。

使用建议：如果只需要某项具体能力（例如 /dcf、/comps），直接装对应的 Vertical Plugin；如果要跑完整流程（如「从草稿到带品牌的 pitch book」），装对应的 Agent Plugin。

9 个端到端 Agent 完整工作流，自带所需 Skills

每个 Agent 是为一类具体岗位设计的：装一个就能跑完整的工作流。下面是高层定位；它们底层调用的 Skill 在后续小节中详细列出。

Pitch Agent

覆盖与咨询：从 comps、precedents、LBO 一直做到带品牌的 pitch deck。

Meeting Prep Agent

客户会议前的简报包 —— 投顾 / WM 场景下用得最多。

Market Researcher

给一个赛道或主题，输出行业概览、竞争格局、可比公司、想法清单。

Earnings Reviewer

电话会 + 财报 → 更新模型 → 草拟研报。

Model Builder

DCF、LBO、三表、Comps，直接在 Excel 中实时构建。

Valuation Reviewer

导入 GP 包，跑估值模板，整理 LP 报告。

GL Reconciler

找出对账差异、定位根因、按流程提交签字。

Month-End Closer

权责发生制、滚动表、差异说明 —— 整套月结。

Statement Auditor

LP 报表派发前做最后一道审计。

第 10 个 Agent KYC Screener 处理入职文档与规则引擎，对应下方 Operations 章节。

Equity Research 9 个 Skill · 股票研究

面向卖方 / 买方研究员的全周期工具：日常晨会 → 财报前后 → 行业深度 → 持仓回顾。

catalyst-calendar

事件追踪

为覆盖股票池维护一份「催化剂日历」：财报日、产品发布、监管决定、宏观事件。

触发："catalyst calendar"、"upcoming events"、"earnings calendar"。

建议在每周一上午跑一次，配合 morning-note 提前布局 1–4 周内的关键事件。

earnings-analysis

财报后

财报已发布的覆盖公司，生成 8–12 页、3000–5000 字的研究更新 —— 含 Beat/Miss 分析、估计调整、论点修正。

触发："earnings update"、"Q1/Q2/Q3/Q4 results"、"post-earnings report"。

输出长度严格控制；适合「24–48 小时内交付」的快速研报场景。需要全面初次覆盖请用 initiating-coverage。

earnings-preview

财报前

财报前定位分析：估计建模、情景框架、关键指标关注列表 —— 设置 Bull / Bear 场景。

触发："earnings preview"、"pre-earnings"、"what to watch for [company]"。

与 earnings-analysis 成对使用：preview 前置 → 财报当日复盘 → analysis 输出。

idea-generation

选股

系统化选股：量化筛选 + 主题研究 + 模式识别，输出新的多 / 空头机会。

触发："stock screen"、"find ideas"、"pitch me something"。

先用 sector-overview 定方向再跑此 Skill，命中率更高；筛出的票直接进 thesis-tracker。

initiating-coverage

深度研报

机构级别的初次覆盖报告，按 5 步工作流执行：公司研究 → 财务建模 → 估值 → 制图 → 报告组装。每步必须单独跑、验证前置依赖。

触发："initiation report"、"initiate coverage on [company]"。

这是仓库里最重的研究 Skill，跑完整 5 步至少要数小时。强制「One Task at a Time」，不要试图一把梭。

model-update

模型维护

把新数据塞进模型 —— 财报实际值、新指引、宏观变化、假设调整。会重算估值并标出实质性变化。

触发："update model"、"plug earnings"、"new guidance"。

财报当晚的标配；和 earnings-analysis 联动使用 —— 一个更新数字、一个出研报。

morning-note

日报

早会用的简短交易日报：隔夜事件 + 交易思路 + 关键日程；为 7:00 AM 晨会量身打造。

触发："morning note"、"morning meeting"、"what happened overnight"。

每个交易日跑一次；最好用 cron 或定时器在 6:30 触发，留半小时人工 review。

sector-overview

行业研究

综合性的行业 / 主题报告：市场规模、竞争格局、核心玩家、主线趋势。深度 5–30 页可控。

触发："sector overview"、"industry deep dive"、"thematic research"。

客户端材料、新覆盖启动、内部知识库都能用 —— 起手要先和用户约定「中性扫描 vs 主题论点」。

thesis-tracker

持仓管理

为持仓 / Watchlist 维护投资论点：跟踪关键数据点、催化剂、里程碑；检查论点是否仍成立。

触发："update thesis for [company]"、"is my thesis still intact"。

每次 earnings-analysis 或重大事件后立刻更新，避免「论点漂移」。

Financial Analysis 13 个 Skill · 财务建模与文档生产

最通用的一个 vertical：估值模型、表格清洗、稿件审计、PPT/Excel 制作 —— 是其他 vertical 的「公共底座」。

3-statement-model

建模

填充三表模型模板（IS / BS / CF），强制使用公式而非硬编码值，并逐步与用户确认每个区块。

触发："fill out 3-statement model"、"populate IS/BS/CF"。

铁律：除「历史值」与「假设驱动」外，所有派生单元格必须是 Excel 公式 —— 这是后续 sensitivity / 场景能跑通的前提。

audit-xls

质检

对电子表格做公式准确性审计 —— 范围可选「单元 / 单 sheet / 整本模型」；后者会做 BS 平衡、Cash tie-out 等完整性检查。

触发："audit this sheet"、"model won't balance"、"debug model"。

DCF / LBO / Merger 模型交付 IC 前的必跑一步；选 "model" 范围才会触发完整体检。

clean-data-xls

数据清洗

清洗脏数据 —— 去空白、统一大小写、文本转数字、统一日期、去重、标记混合类型列。

触发："clean this data"、"normalize this data"、"dedupe"。

放在分析 / 建模流水线的最前面；保留原数据为「helper column」模式，便于回溯。

competitive-analysis

竞品

两阶段竞品 deck：先 scope（公司圈定、深度、是否投资语境），再生产 —— 涉及市场地图、对手深扒、对比分析、战略综合。

触发："competitive landscape"、"benchmark X against peers"、"market map"。

「投资 memo」和「战略评估」两种深度差别很大，开始前一定要先和用户约定 scope。

comps-analysis

估值

机构级别的可比公司分析：运营指标 + 估值倍数 + 统计基准。强制走数据源优先级：MCP（Kensho / FactSet / Daloopa）> 机构来源 > 公开搜索。

触发："comps"、"trading multiples"、"peer valuation"。

对私有公司、综合性集团、困境企业、Pre-revenue 启动器 —— 不要硬用，应换 DCF 或场景估值。

dcf-model

估值

投行级 DCF 模型：从 SEC 文件拉数据、做现金流预测、WACC、敏感性分析，输出含执行摘要的 Excel 模型。

触发："DCF model"、"intrinsic value"、"terminal value"。

Office JS / Python 双环境支持 —— 在 Excel 加载项里跑用 Office JS，离线生成 .xlsx 用 openpyxl + recalc。

deck-refresh

PPT 更新

用新数字「刷新」现有 deck —— 季度更新、财报更换、Comps roll、市场数据 rebase。保留原格式，只换数值。

触发："update the deck with Q4 numbers"、"refresh the comps"、"swap in the new earnings"。

强制 4 阶段，第 3 阶段是用户审批 gate —— 不要在用户看到 mapping 前去改 deck。

ib-check-deck

QC

投行 deck 的最终质检：跨 slide 的数字一致性、数据与叙述对齐、IB 语言规范、视觉与格式。只读不改。

触发："check my deck"、"is this client-ready"、"reconcile figures across slides"。

发给客户前的最后一道关卡；它不修改，你拿到报告后再决定改不改。

lbo-model

PE 建模

完整填充 LBO 模板：Sources & Uses、运营模型、Debt Schedule、Returns Analysis；遵循专业格式标准。

触发："LBO model"、"leveraged buyout model"、"sponsor returns"。

永远优先用用户给的模板 —— 哪怕模板看起来繁琐，也不要「从零构建」。

ppt-template-creator

元 Skill

「Skill 的 Skill」—— 把用户的 PowerPoint 模板转成一个独立的可复用 Skill；不是用来直接出 deck。

触发："turn this template into a skill"、"make a reusable skill from my .pptx"。

只在希望长期复用模板时用；出一张实际 deck 请用 pptx-author 或 pitch-deck。

pptx-author

无头 PPT

在 managed-agent / CMA 模式下，没有 Office App 时生成 .pptx 文件 —— 写到 ./out/<name>.pptx。

触发：headless 环境下任何需要 PPT 输出的请求。

基础设施 Skill；和 xlsx-author 是一对。

skill-creator

元 Skill

教如何写 Skill 的 Skill —— 用于新建或更新 Skill 文件结构、SKILL.md 规范、上下文管理原则。

触发："create a new skill"、"author a skill"。

扩展这个仓库时的起点；强调「context window 是公共资源」、"concise is key"。

xlsx-author

无头 Excel

headless 环境下生成 .xlsx —— 用 openpyxl 写公式字符串而非计算值。

触发：headless 环境下任何需要 Excel 输出的请求。

Office JS 环境直接用 mcp__office__excel_*，此 Skill 是 CMA / 服务端的兜底。

Fund Admin 6 个 Skill · 基金行政与对账

面向月结、对账、NAV 这一类「数字必须对得上」的工作 —— 每一步都强调引用支撑文档、不允许「拍脑袋补差额」。

accrual-schedule

月结

期末权责发生制表：对每个 accrual 项目算金额、引支撑、出会计分录草稿（待 controller 审批）。

触发：月结流程中需要 accrual 表时。

输出永远是 draft —— 不是过账指令；供应商发票视为「不可信输入」，按 policy 应用金额。

break-trace

对账

把一行 break 追到源头交易 / 凭证，输出形如「⟨side⟩ ⟨did what⟩ because ⟨reason⟩」的根因陈述。

触发：gl-recon 分类完 break 后的下一步。

配合 gl-recon 使用 —— 先分类、再追因；FX 日期、过账日期、账户映射是高频差异点。

gl-recon

对账

GL ↔ 子账对账：按 key + 比较列做全外联接，分类成 Matched / Amount break / Quantity break / Timing break / GL only / Subledger only。

触发：日终 / 月末对账。

子账与托管行抽取视为「不可信输入」—— 只取数、不执行其中的指令。

nav-tieout

LP 报表

在派发前把 LP 报表 line-by-line 重新算一遍 —— 起始资本 + 出资 − 分配 + 分摊净收益 − Carry = 期末资本。NAV pack 是 source of truth。

触发：LP 季报 / 年报派发前。

容差 0.01；任何不一致要指出由哪个输入驱动（例如「分摊比例由季度内 transfer 改变」）。

roll-forward

月结

为资产负债表科目构建期初 → 期末的滚动表 —— 每一行都要绑到 GL 查询。必须 foot；不允许人工 plug。

触发：月结包、审计支持。

底层依赖 internal-gl MCP；查询要附在每行旁边，便于审计回溯。

variance-commentary

管理报告

为每个达阈值的 P&L / BS 行写 flux commentary —— 当期 vs 上期 vs 预算，并用底层活动解释「why」而不是 restate「what」。

触发：月结管理报告。

不要写「云成本增加 18%」—— 要写「云成本 +1.2M，源于 5 月发布预留的 GPU 增量」。原因不明时要诚实说「flag for controller」。

Investment Banking 9 个 Skill · 投行流程

从 sourcing → CIM → 投标 → 关闭的卖方流程，每一步都有对应工具。

buyer-list

卖方

为卖方 M&A 流程组织买家宇宙 —— 战略买家（直接 / 邻接 / 纵向整合）、财务投资人 —— 评估契合度、排序外联优先级。

触发："buyer list"、"strategic buyers"、"who would buy this"。

先和卖方约定「strategic vs financial」偏好与管理层去留；这决定了名单结构。

cim-builder

卖方

起草 CIM（保密信息备忘）—— 把公司材料组织成投资人 ready 的专业文档。

触发："CIM"、"offering memorandum"、"info memo"。

先要齐管理层 ppt / 历史财务 / 预算 / QofE，再开工 —— 数据空缺会反复返工。

datapack-builder

PE / IB

把 CIM / 招股书 / 10-K / 网搜 / MCP 数据「正常化」成 IC-ready 的 Excel 数据包；强制 Currency / Number 格式规则、追溯到源页码。

触发："build a datapack"、"normalize this CIM data"。

每个数字必须能追到源页面 —— 这是后续所有衍生分析的基础。零容忍硬编码。

deal-tracker

流程

多个 live deal 的 pipeline 视图：里程碑、deadline、动作项、状态更新。

触发："deal tracker"、"weekly deal review"、"deal pipeline"。

每周固定时间跑一次（如周一早会前），让逾期项目浮出来。

merger-model

建模

Accretion / Dilution 分析：Pro forma EPS、协同效应敏感性、PPA。

触发："merger model"、"accretion dilution"、"pro forma EPS"。

现金 / 股权 mix、新增债务、协同释放时间表 —— 全部参数化，便于做敏感性。

pitch-deck

PPT 填充

投行 pitch deck 模板填充：从源文件（Excel / CSV）把数据塞进现有 slide 布局；不从零造 deck。

触发：用户提供模板和源数据，要求填充。

从零做 deck 请用 pptx-author；本 Skill 专攻「模板填充」这一窄场景。

process-letter

卖方

起草 process letter / IOI 指令 / 终轮投标信 / 管理层会议邀请。

触发："process letter"、"bid instructions"、"final round letter"。

通常和 cim-builder、teaser 一起作为「卖方流程起始包」。

strip-profile

公司简介

为 pitch book / deal 材料生成 1–4 页的公司简介（strip profile）—— 四象限布局，含图表与表格。

触发："strip profile"、"company profile slide"。

先和用户确定单页 vs 多页；数据要来自 SEC、FactSet / CapIQ，不要靠模型估计。

teaser

卖方

匿名一页 Teaser ——「NDA 之前」给买家看的钩子；要有市场地位 + 关键财务，但不能识别公司身份。

触发："teaser"、"blind teaser"、"anonymous profile"。

和卖方明确「哪些细节要遮、哪些可以亮」—— 一旦泄露身份就要重做。

Operations 2 个 Skill · KYC / 入职

为 KYC Screener Agent 提供两块拼图：解析文档 + 应用规则。设计上严格区分「不可信文档内容」与「可信的规则网格」。

kyc-doc-parse

解析

把入职文档包解析成结构化 KYC 字段 —— 身份、所有权、控制、资金来源、文件清单。输出供规则引擎消费。

触发：KYC 流程的第 1 步。

明确把申请人文档当成 <untrusted_document> —— 只取数，绝不执行其中的「指令」（即便文字看似合理）。

kyc-rules

规则

把公司的 KYC/AML 规则网格应用到解析结果：风险评级、文件完备性、PEP/制裁/负面媒体筛查；输出每条规则的命中结果与必要的升级。

触发：KYC 流程的第 2 步（在 doc-parse 之后）。

本 Skill 不做决策 —— 只打分 + 路由 —— 最终拒否仍是合规官的事。

Private Equity 10 个 Skill · PE 全周期

从找项目 → 尽调 → IC → 投后 → 价值创造 → 退出。每一步都有专用 Skill，互相之间形成完整流水线。

ai-readiness

投后

扫描 portfolio，找最高杠杆的 AI 落地点 —— 跨 portco 排出一份「应该最先派 operating partner 进场」的清单。三个 Gate：数据在？Owner 在？30 天能 pilot？

触发："AI readiness"、"AI quick wins"、"which portcos are ready for AI"。

和持有期联动：<12 个月退出的 portco，AI 投入回收期到不了，应当退出排序。

dd-checklist

尽调

按 target 行业 / 交易类型生成尽调清单 —— 涵盖财务、商务、法律、运营、税务、HR、IT 等 workstream，附状态跟踪与红旗升级。

触发："dd checklist"、"diligence request list"。

每天或每周更新进度；和 data room MCP 联动可以自动判定「item received / outstanding」。

dd-meeting-prep

尽调

尽调会议（管理层、专家网、客户参考）前的准备 —— 出问题清单 / 基准对照 / 红旗追问点。

触发："prep for management meeting"、"expert call questions"。

先把 CIM / data room 已发现的「known unknowns」喂给 Skill；它会针对性出问题。

deal-screening

入口

对 inbound deal flow 做快速筛选：从 CIM / teaser 抽要素，套基金投资标准，输出一页筛选 memo。

触发："screen this deal"、"should we look at this"。

先和基金一次性约定 investment criteria（营收 / EBITDA / margin / 增长曲线），后续筛选才有意义。

deal-sourcing

入口

三步 sourcing：发现公司 → CRM 查重 → 起草个性化创始人外联。

触发："source deals"、"draft founder email"。

CRM 查重要先做 —— 否则容易给已经 pass 过的公司再发外联，损害关系。

ic-memo

投决

起草 IC memo —— 综合尽调发现、财务分析、交易条款，输出结构化、IC-ready 的推荐文档。

触发："IC memo"、"deal write-up"、"recommendation memo"。

把前面的 dd-checklist、returns-analysis、value-creation-plan 输出全部喂入；不要让 IC memo 重新做分析。

portfolio-monitoring

投后

投后季度 / 月度跟踪：吃 portco 财务包，抽 KPI，做 budget variance，输出 dashboard；含 covenant 检查。

触发："review portfolio company"、"covenant check"。

设一个固定的 KPI 集合 —— 否则不同 portco 报告口径不一，无法做 portfolio 级聚合。

returns-analysis

建模

快速 IRR / MOIC 敏感性 —— 入口 multiple、杠杆、出口 multiple、增长、持有期 5 维 sensitivity。

触发："returns analysis"、"IRR sensitivity"、"MOIC table"。

投决会前出几个 scenario：Base / Upside / Downside —— IC 喜欢看的是「最差情况下还能拿多少」。

unit-economics

尽调

SaaS / 订阅 / 经常性收入业务的单位经济分析：ARR Bridge、Cohort、LTV/CAC、Net Retention、Payback、Revenue Quality。

触发："unit economics"、"cohort analysis"、"LTV CAC"。

对 enterprise / SMB / mid-market 要分段算 —— blended 数会掩盖结构性问题。

value-creation-plan

投后

post-close 价值创造规划：把 revenue / cost / 运营杠杆 mapping 到 EBITDA Bridge；含 100-day 优先级、KPI、问责框架。

触发："value creation plan"、"100-day plan"、"EBITDA bridge"。

入场前的 VCP 草稿 ↔ 投后 6 个月再更新 —— 与 portfolio-monitoring KPI 对齐。

Wealth Management 6 个 Skill · 财富管理

面向高净值客户顾问的全周期：从首次提案 → 季度回顾 → 财务规划 → 节税 → 再平衡 → 客户报告。

client-report

报告

面向客户的季度 / 年度业绩报告：组合收益、配置拆解、市场评论 —— 含品牌与披露。

触发："client report"、"quarterly report for [client]"。

benchmark 要和 IPS 对齐；同一客户不要中途切换 benchmark。

client-review

会议准备

客户回顾会准备包：业绩 + 配置 + 谈话要点 + 动作项 —— 把账户数据汇成一份会议 ready 的格式。

触发："client review"、"meeting prep for [client]"。

会前找到上次会议留下的 action items；漏跟会让客户觉得「他们只是来收费的」。

financial-plan

规划

综合财务规划：退休投影、教育金、遗产、现金流 —— 用于新客户 onboarding、年度复审、情景建模。

触发："financial plan"、"can I retire"、"education funding"。

通胀 / 税率假设要明确写出来，便于后续年度更新对比。

investment-proposal

获客

给潜在客户的投资提案：公司介绍、配置建议、预期收益、费率结构。

触发："investment proposal"、"prospect presentation"。

「Understanding Your Needs」一节要复述客户讲过的话 —— 这一页决定了客户是否觉得被听到。

portfolio-rebalance

交易

分析配置漂移并生成跨账户的再平衡交易建议 —— 考虑税务影响、交易成本、wash sale 窗口。

触发："rebalance"、"portfolio drift"、"allocation check"。

「先在 IRA/Roth 里再平衡，再在 taxable 里动」—— 默认走税务最优路径。

tax-loss-harvesting

节税

在 taxable 账户里找 TLH 机会：未实现亏损 → 建议替代标的 → 跟踪 wash sale。

触发："TLH"、"harvest losses"、"year-end tax planning"。

Q4 月底前是密集期；替代标的要确保不构成 wash sale（不是「实质相同」）。

LSEG 合作 Skill 8 个 Skill · 固收 / FX / 利率

由 LSEG（伦敦证券交易所集团）提供 —— 全部基于其 MCP 工具链（bond_price、interest_rate_curve、fx_vol_surface 等）。Skill 不自己算数，工具算、Skill 负责编排和综合。

bond-futures-basis

Partner

债券期货基差分析 —— 定价期货、识别 CTD、对比收益曲线，评估交割期权价值与基差交易机会。

核心：计算 Net Basis 与 Implied Repo，与市场 Repo 对比判断期货 rich / cheap。

需要 LSEG MCP 工具组连接；适合做空 / 多基差头寸的策略台。

bond-relative-value

Partner

债券相对价值分析 —— 总价差分解（无风险 + 信用 + 残差）、场景压力测试。

核心：残差价差 = G-spread − Credit curve spread，揭示真正的 rich / cheap。

配合 fixed_income_risk_analytics，可深度处理含权债。

equity-research

Partner

股票研究快照 —— IBES 一致预期 + 公司基本面 + 历史价格 + 宏观背景，输出结构化研究 snapshot。

核心问题："where might consensus be wrong"。

和上文 Anthropic 的 equity-research vertical 是不同的 —— 这里走 LSEG / IBES 数据；适合需要 IBES 一致预期的场景。

fixed-income-portfolio

Partner

固收组合分析：批量定价 → 加权聚合（yield / duration / DV01）→ 现金流瀑布 → 标准 rate shock 场景。

核心：市值加权的 portfolio 级指标 + 单券的 reference data 拆分。

每次 portfolio review 跑一次；标准 shock（±50 / ±100 / ±200 bp）是 IC / 风控的通用语言。

fx-carry-trade

Partner

FX Carry 评估 —— Spot + Forward Curve + Vol Surface + 历史趋势；核心指标是 Carry-to-Vol 比率。

核心：annualized carry / ATM implied vol。

Carry 本质是 short volatility —— Vol 上升是首要风险信号，注意监控 ATM 与 Risk Reversal。

macro-rates-monitor

Partner

宏观与利率 dashboard：周期位置（GDP/PMI）、央行（policy rate / curve）、债市（slope / real rate）、金融条件（swap spread）。

核心：把多个指标融合成一段「叙事」。

每周一上午跑一次；与 swap-curve-strategy 形成「宏观 → 曲线策略」的下游链路。

option-vol-analysis

Partner

期权波动率分析 —— Vol Surface（ATM 期限结构 / Risk Reversal / Butterfly）、Greeks、Implied vs Realized。

核心：vol premium = implied − realized，判断期权贵不贵。

先看 surface 形状，再为特定 strike 计算 Greeks —— 不要反过来。

swap-curve-strategy

Partner

利率掉期曲线策略：构建 swap 曲线 → 叠加政府曲线（swap spread）→ 通胀曲线（real rate decomp）→ 计算 2s10s / 5s30s / butterfly。

核心：给出 DV01 中性的 steepener / flattener / butterfly 交易思路。

所有交易建议都要附 DV01 中性的 sizing 和 carry / roll-down —— 别只给方向不给量。
