Crusoe 是 2018 年由 Chase Lochmiller（CEO）和 Cully Cavness（President）创立的「AI 工厂公司」。原本通过 Digital Flare Mitigation 技术把油气田废气转成比特币挖矿算力，2024 年底剥离比特币矿业后完全转向 AI 算力。公司的核心叙事是「垂直一体化的 AI 工厂」—— 自己拿能源、自建数据中心、自己制造组件、自己提供 GPU 云，并以「BYO Power」（自带电力）模式绕过传统电网瓶颈。

核心机会：在 SemiAnalysis ClusterMAX 排名中位列 Gold tier；是 OpenAI Stargate 项目 Abilene 旗舰站点（1.2 GW）的开发方；2025 H1-Q3 Cloud bookings 同比 5x，电力 pipeline 一年内增长 4 倍至 45 GW。
核心风险：2026 年 3 月 OpenAI/Oracle 取消 Abilene 600 MW 扩建，理由包括「冬季液冷故障导致多日停机」损害了关系、OpenAI 想等 Vera Rubin 芯片；Crusoe 正在拉 Meta（在 Nvidia 撮合下）和 Microsoft 接盘 —— 这是单一大客户集中度风险的明确信号。

① AI 算力 / Neocloud 赛道概览 sector-overview

AI 推训需求在 2024–2026 年快速从「应急买电」演变成「战略基础设施合同」。「Neocloud」（新云）—— 以 GPU-as-a-Service 为单一焦点的厂商 —— 正在以高速互连、专门化和价格三个维度争夺 Hyperscaler 的份额，而 Hyperscaler 反过来也开始把它们当成「容量基础设施」做长期锁定（如 CoreWeave–Meta $21B、Nebius–Microsoft $17.4B）。

市场结构：三类玩家

类别典型玩家定位
HyperscalersAWS / Azure / GCP / Oracle多服务组合；GPU 是其中一项；走 sticky 客户与 enterprise 合规护城河
NeocloudsCoreWeave、Crusoe、Lambda、Nebius、Together AI纯 GPU 云；价格 + 技术专业化竞争；与 Hyperscaler 既是替代也是供应商
长尾Vast.ai、RunPod、SF Compute、Hyperstack 等极致低价；适合预算敏感的研究与中小客户

5 条主线趋势

  1. 电力是真正的瓶颈 —— 单机柜功耗从 15 年前的 2–4 kW 升到 AI 时代的 140 kW。建数据中心的难点从「征地 + 机房」变成「能在 18 个月内拿到 100MW–1GW 电力的能力」。Crusoe 的核心论点正是这一条。
  2. Hyperscaler 反向锁定 Neocloud 容量 —— Meta 与 CoreWeave 的 $21B、Microsoft 与 Nebius 的 $17.4B、Meta 与 Nebius 的 $3B/5 年 ——「Reserved capacity 合同更像 dark fiber 预订或半导体长协，而非普通云开销」。
  3. Stargate 等大基建拉动需求 —— OpenAI / Oracle / SoftBank / MGX 的 $500B Stargate 是赛道最大需求方，但需求侧的灵活性极高（OpenAI 可以在 Vera Rubin 准备好时换站点，参见 Abilene 取消事件）。
  4. 价格在 race-to-the-bottom —— H100 时薪在 2024–2025 年从约 $8 跌至 $1.45–$6.15 区间，4x 价差，「不可持续」。这倒逼厂商寻找差异化（垂直整合、自有电力、模块化、地理 / 主权数据）。
  5. 架构理念分裂：裸金属 vs 轻量 VM —— CoreWeave、Oracle 走裸金属 + DPU（NVIDIA BlueField）；Nebius、Crusoe 走轻量 VM（cloud-hypervisor），换 seconds-level 配置 / snapshot / 隔离。Nebius 已证明 VM 路线可达裸金属性能；Crusoe 仍在收敛中。

② 竞品对比 competitive-analysis

战略象限：规模 vs 差异化

规模优先（容量基础设施）

  • CoreWeave —— 已经是 Hyperscaler 的容量供应商
  • Crusoe —— 通过 Stargate + 垂直整合争夺这块

赢的方式：拿到最便宜的电、用 long-term 合同锁客户、用产能换溢价。

差异化优先（专门化平台）

  • Nebius —— EU 主权 + VM 路线 + Hyperscaler 主备角色
  • Together AI —— 模型即服务体验
  • Lambda —— 易用 + 研究人员心智

赢的方式：在某一垂直 / 客户层做到 sticky；不和 CoreWeave 拼绝对规模。

Crusoe 的相对位置

Crusoe 同时在抢规模和差异化两条线。规模线（Stargate / 1.8 GW Wyoming）让它和 CoreWeave 直面竞争；差异化线（清洁能源 + 模块化 Spark + Edge Zones + 太空数据中心）让它和单一规模玩家拉开。

潜在问题：这两条线消耗的资本性质不同 —— 规模需要长期债务（Brookfield $750M、Goldman $300M），差异化需要 R&D / M&A（收购 Atero、与 GE Vernova / Redwood Materials / Starcloud 合作）。能否两边都做透是核心估值变量。

商业模式：垂直一体化 AI 工厂

传统 AI 基础设施是「四段式」—— 发电公司 + 地产开发商 + 数据中心运营商 + 云平台 —— 步调不一致、激励错配、周期长。Crusoe 把这 4 段合到一家公司：

价值链环节Crusoe 怎么做
① 能源「BYO Power」—— 自己拿停滞 / 可再生能源，与 GE Vernova（燃气轮机）、Redwood Materials（二代 EV 电池）、Tallgrass、Lancium 合作；电力 pipeline 已达 45 GW
② 选址与开发跟着能源走，而非跟着电网走 ——「Move the compute to the energy」
③ 数据中心组件制造Crusoe Industries 国内自产；Spark Factory 生产模块化数据中心，可在 3 个月内立一个 Edge Zone
④ 云平台Crusoe Cloud（GPU 出租）+ Crusoe Managed Inference（MemoryAlloy 内存优化技术）

产品线

Crusoe Cloud

核心 GPU 云：H100 / H200 集群；2025 H1–Q3 bookings 同比 +5x；99.98% 上线率；面向 AI 训练 + 推理工作负载；Iceland / Nordic 站点支持低 PUE。

Crusoe Managed Inference

2025 推出；基于自研推理引擎 + MemoryAlloy 内存优化。收购 Atero（GPU 内存优化创业公司）后能力得到补强。

Crusoe Spark + Edge Zones

2026/03 推出 Edge Zones。模块化数据中心单元，主权 / 低延迟 AI 部署；可在传统 Hyperscaler 和 Neocloud 不到的地点立刻部署。

Crusoe Industries

国内制造端 —— 把数据中心组件从「供应链外购」变成「自家可控」。是公司 BYO Power 策略的硬件侧。

核心结论与监控指标

三个核心论点

  1. Crusoe = 「能源套利 + 垂直一体化 AI 工厂」。它不是单纯的 GPU 出租商，而是 BYO Power 模式下唯一能在 18 个月内把 1 GW+ 能源转化成在线 AI 工厂的玩家。这一点是其相对 CoreWeave / Lambda 等纯算力玩家的根本差异化。
  2. 当前估值由 Stargate 锚定，但 Stargate 自己也在裂痕中。OpenAI 已展示出会切换站点 / 等下一代芯片的灵活性；好消息是需求过剩到 Microsoft 立刻接盘 —— 但「替代客户接盘的折扣率」、「不同 Hyperscaler 之间的迁移成本」会决定下个估值锚定点。
  3. 管理层 + 投资人结构已经在为 IPO 排队。Michael Gordon（前 MongoDB COO/CFO）、Goldman Sachs 债务首投、Crossover 基金在 Series E 大举进入 —— 2027–2028 IPO 是基本情形。
